# P5 — Site Foundation (Astro) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **BEFORE you touch any visual task (T1, T3, T4, T5):** invoke the `frontend-design` skill (Skill tool) and re-read §"Design system" below. The aesthetic — **terminal × editorial** — *is the product*. The founder wants "one of the most beautiful" sites on the web. Generic AI-template output (Inter on white, purple gradients, evenly-distributed timid palettes, cookie-cutter card grids) is a FAILURE of this plan even if it builds. Execute the locked design tokens in T1 with precision; do not invent your own palette.

**Goal:** A static Astro app `apps/site` (`@khazana/site`) that renders khazana's three surfaces — **Feed** (`/`), **Reads** (`/reads`, `/reads/[slug]`), **Workshop** (`/workshop`) — over a committed sample feed and a `blog` MDX content collection, wrapped in a distinctive terminal-chrome shell with editorial long-form reading. Includes a taste-capture beacon (no-op without a worker URL) and PWA basics. Builds with **NO** generated data and **NO** `PUBLIC_WORKER_URL`. The heavy interactive MDX component library, ⌘K palette, full-text search, connections graph, and taste dashboard are **P5B** — this plan leaves clean SEAMS for them, not implementations.

**Architecture:** Pure data helpers live in `src/lib/` and are unit-tested with vitest under strict TDD (failing test → run/FAIL → implement → run/PASS → commit); they take an injected `now` and never touch the network. Astro components (`.astro`) and CSS are not unit-tested — those tasks are structured as *write complete code → verify with `pnpm --filter @khazana/site build` (astro build succeeds) → commit*. The site depends on `@khazana/core` for all types/vocabulary (`FeedItem`, `FeedItemSchema`, `CHANNELS`, `FORMAT_NAMES`, `FORMATS`) and on `@khazana/worker`'s browser client (`sendEvent`/`getDeviceId`) for the beacon — neither is redefined. Astro static output with a configurable `site`/`base` supports GitHub Pages project pages.

**Tech Stack:** Astro 5 (`output: "static"`), `@astrojs/mdx`, TypeScript 5 (strict, ESM, `verbatimModuleSyntax`), vitest 2, zod (via `@khazana/core`), `@khazana/core` + `@khazana/worker` (workspace). No UI framework in the shell; islands are deferred to P5B. System/self-hostable fonts only ($0).

## Global Constraints

*(Copied verbatim from the P5 brief — treat as hard gates.)*

- **$0:** static GitHub Pages output; local/system fonts only; no paid services; build works with NO generated data (sample fallback) and NO worker URL (beacon no-ops).
- **Aesthetic bar:** terminal×editorial, distinctive, NOT generic AI template. Invoke `frontend-design` skill.
- **Reuse `@khazana/core` types/vocab** (FeedItem, FORMATS, CHANNELS); never redefine. Beacon via `@khazana/worker` client.
- **Accessibility:** semantic HTML, keyboard-navigable, `prefers-reduced-motion`/contrast respected.
- **pnpm; ESM; TS strict.** Leave clean seams (⌘K, search, interactive components) for P5B — placeholders, not implementations.

**Additional encoded decisions (from brief + repo facts):**

- Astro config must read `site`/`base` from env (`PUBLIC_SITE_URL`, `PUBLIC_BASE_PATH`) with sane defaults so GitHub Pages project-page paths work without code edits.
- `@khazana/worker` does **not** re-export its browser client from its package `main` (that's the Worker fetch handler). Import the client from its module subpath: `import { sendEvent, getDeviceId } from "@khazana/worker/src/client.ts"`. Do **not** add a new export map to the worker package in this plan; the subpath import keeps P5 isolated.
- The generated feed `data/feed/curated.json` is **gitignored** (`data/feed/` is in `.gitignore`). The loader must fall back to a committed `data/feed/curated.sample.json` and validate every item with `FeedItemSchema`.
- `vitest.config.ts` already includes `apps/**/*.test.ts` — `src/lib` tests under `apps/site` are picked up by the root `pnpm test` automatically. No config change needed.
- ESM rule: relative imports use `.js` extensions; type-only imports use `import type` / inline `type` (`verbatimModuleSyntax` is on).

## Locked Design System (terminal × editorial)

> This is the authoritative visual contract. T1 encodes it verbatim into `src/styles/tokens.css` + `src/styles/global.css`. Do not substitute fonts or colors.

**Concept.** A refined *signal terminal* — mono chrome, hairline rules, a live phosphor ticker, bracketed section labels `[ FEED ]`, a blinking block cursor on the wordmark — that opens into *Distill/Pudding-grade* long-form reading: warm serif, generous measure, margin citations. Dark-first. One terminal accent (phosphor amber), one editorial accent (clay/rust). Restraint over decoration; the cursor blink, the hairlines, and the reading typography carry the identity.

**Palette (CSS custom properties, dark default):**

| Token | Value | Role |
|---|---|---|
| `--bg` | `#0a0a0b` | near-black page background |
| `--bg-raised` | `#111113` | cards, raised surfaces |
| `--bg-inset` | `#070708` | ticker / inset wells |
| `--ink` | `#e8e6df` | warm off-white primary text |
| `--ink-dim` | `#a3a097` | secondary text |
| `--ink-faint` | `#6a6862` | mono labels, metadata |
| `--rule` | `#26252b` | hairline dividers/borders |
| `--rule-bright` | `#3a3942` | hover/focus rule |
| `--accent` | `#ffb627` | phosphor amber — terminal accent (links, cursor, ticker prompt) |
| `--accent-dim` | `#b8851d` | amber pressed/visited |
| `--editorial` | `#c1554a` | clay/rust — editorial accent (format badges, article rules) |
| `--editorial-dim` | `#8f3c34` | clay pressed |
| `--good` | `#7faa6e` | sage — positive metric (high trust) |
| `--focus` | `#ffb627` | focus ring color |

Light mode (only if `prefers-color-scheme: light` AND no explicit dark): invert to `--bg:#f4f2ec; --bg-raised:#fbfaf6; --ink:#1a1916; --ink-dim:#52504a; --ink-faint:#8a877e; --rule:#dcd9cf; --accent:#b8851d; --editorial:#a8463c;`. Dark is the default and the designed-for path; keep light correct but secondary.

**Type stacks (NO paid fonts; system-first, self-hostable later):**

- `--font-mono`: `"Berkeley Mono", "JetBrains Mono", "SF Mono", ui-monospace, "IBM Plex Mono", Menlo, monospace` — ALL chrome, labels, badges, metrics, ticker, wordmark.
- `--font-read`: `"Iowan Old Style", "Charter", "Palatino Linotype", "Palatino", "Source Serif 4", Georgia, serif` — Reads long-form body + article titles.
- `--font-sans`: `"Söhne", ui-sans-serif, system-ui, -apple-system, "Helvetica Neue", sans-serif` — used sparingly (Feed/Workshop card titles where mono is too tight); chrome stays mono.

> Self-host seam (P5B, optional): if a free variable serif is later added, drop a `@font-face` woff2 into `src/styles/fonts/` and prepend it to `--font-read`. Do NOT add font files in P5.

**Type scale (1.25 ratio, rem):** `--t-xs:0.75rem; --t-sm:0.8125rem; --t-base:1rem; --t-md:1.125rem; --t-lg:1.375rem; --t-xl:1.75rem; --t-2xl:2.25rem; --t-3xl:3.25rem;`
**Reading measure:** `--measure: 68ch;`  **Line heights:** `--lh-tight:1.15; --lh-read:1.62;`

**Spacing scale (8px base):** `--s-1:0.25rem; --s-2:0.5rem; --s-3:0.75rem; --s-4:1rem; --s-5:1.5rem; --s-6:2rem; --s-8:3rem; --s-10:4rem; --s-12:6rem;`

**Radii & misc:** `--r-sm:2px; --r-md:4px;` (terminals are crisp — small radii only). `--hair:1px;` (hairline width). `--maxw:1180px;`

**Signature motifs (must appear):**
1. **Blinking block cursor** after the `khazana` wordmark — a `▋` that blinks via `@keyframes blink`, frozen open under `prefers-reduced-motion`.
2. **Hairline rules** (`--rule`) separate every chrome region; no heavy borders, no drop shadows on chrome.
3. **Bracketed mono section labels:** `[ FEED ]`, `[ READS ]`, `[ WORKSHOP ]` in `--font-mono`, uppercase, `letter-spacing: 0.08em`, `--ink-faint`.
4. **Live ticker** with a leading `›` phosphor prompt and `·` separators; CSS marquee, pauses on hover/focus, fully static under reduced motion.
5. **Phosphor glow** on accent hover only: `text-shadow: 0 0 8px color-mix(in oklab, var(--accent) 50%, transparent)`.
6. **Faint scanline/grain overlay** on `body::after` at `opacity: 0.025` (repeating-linear-gradient), disabled under reduced motion and in light mode.

**Reusable primitives (built in T1 + T3):** `ChannelChip`, `SourceBadge`, `FormatBadge`, `Metric`, `TimeAgo`, `Rule` (thin divider). All mono, all token-driven.

```ts
const MIN = 60_000;
const HOUR = 60 * MIN;
const DAY = 24 * HOUR;
const YEAR = 365 * DAY;

/** Compact relative time, e.g. "now", "30m", "3h", "2d", "1y". `now` injected for determinism. */
export function timeAgo(iso: string, now: Date): string {
  const then = Date.parse(iso);
  const delta = now.getTime() - then;
  if (Number.isNaN(then) || delta < MIN) return "now";
  if (delta < HOUR) return `${Math.floor(delta / MIN)}m`;
  if (delta < DAY) return `${Math.floor(delta / HOUR)}h`;
  if (delta < YEAR) return `${Math.floor(delta / DAY)}d`;
  return `${Math.floor(delta / YEAR)}y`;
}

/** "YYYY-MM-DD HH:mm" in UTC, for the footer build stamp. */
export function formatBuildTime(iso: string): string {
  return iso.slice(0, 16).replace("T", " ");
}
```

`apps/site/src/lib/feed.ts`:
```ts
import { existsSync, readFileSync } from "node:fs";
import { join } from "node:path";
import { FeedItemSchema, type FeedItem } from "@khazana/core";

const WORKSHOP_CHANNELS = new Set([
  "ideas",
  "diy",
  "3d-printing",
  "iot",
  "embedded",
  "ai-projects",
]);

/**
 * Load the curated feed. Prefers the pipeline-generated `curated.json`
 * (gitignored), falling back to the committed `curated.sample.json` so the
 * site always builds. Each item is validated with FeedItemSchema; invalid
 * items are dropped (never crash the build). Curated order is preserved.
 */
export function loadCurated(dataDir: string): FeedItem[] {
  const main = join(dataDir, "curated.json");
  const sample = join(dataDir, "curated.sample.json");
  const path = existsSync(main) ? main : sample;
  if (!existsSync(path)) return [];
  const raw: unknown = JSON.parse(readFileSync(path, "utf8"));
  if (!Array.isArray(raw)) return [];
  const out: FeedItem[] = [];
  for (const entry of raw) {
    const parsed = FeedItemSchema.safeParse(entry);
    if (parsed.success) out.push(parsed.data);
  }
  return out;
}

/** Items whose `topics` include the channel. `null`/empty channel → all items. */
export function filterByChannel(items: FeedItem[], channel: string | null): FeedItem[] {
  if (!channel) return items;
  return items.filter((it) => it.topics.includes(channel));
}

/** Workshop selector: kind=idea OR any buildable-channel topic. */
export function selectIdeas(items: FeedItem[]): FeedItem[] {
  return items.filter(
    (it) => it.kind === "idea" || it.topics.some((t) => WORKSHOP_CHANNELS.has(t)),
  );
}

/** First `n` item titles, for the shell ticker. */
```

`apps/site/src/lib/track.ts`:
```ts
import { sendEvent } from "@khazana/worker/src/client.ts";

type SendFn = typeof sendEvent;

export interface TrackDeps {
  endpoint: string;
  send?: SendFn;
  now?: string;
  deviceId?: string;
}

/** Read PUBLIC_WORKER_URL from an env-like object; "" when unset (→ beacon no-ops). */
export function resolveEndpoint(env: { PUBLIC_WORKER_URL?: string }): string {
  return (env.PUBLIC_WORKER_URL ?? "").trim();
}

function fire(
  body: { itemId: string; type: "open" | "dwell"; dwellMs?: number },
  deps: TrackDeps,
): boolean {
  if (!deps.endpoint) return false; // no endpoint → graceful no-op
  const send = deps.send ?? sendEvent;
  return send(deps.endpoint, body, { now: deps.now, deviceId: deps.deviceId });
}

/** Fire an `open` event for an item. No-ops without an endpoint. */
export function trackOpen(itemId: string, deps: TrackDeps): boolean {
  return fire({ itemId, type: "open" }, deps);
}

/** Fire a `dwell` event (with dwellMs) for an item. No-ops without an endpoint. */
export function trackDwell(itemId: string, dwellMs: number, deps: TrackDeps): boolean {
  return fire({ itemId, type: "dwell", dwellMs }, deps);
}
```

```astro
---
// Endpoint is resolved at build time and inlined; empty → the script no-ops.
const endpoint = (import.meta.env.PUBLIC_WORKER_URL ?? "").trim();
const isRead = Astro.url.pathname.includes("/reads/") && !Astro.url.pathname.endsWith("/reads/");
---

<script is:inline define:vars={{ endpoint, isRead }}>
  // Taste-capture beacon. No framework, no deps at runtime: we re-implement the
  // tiny sendBeacon call inline so this stays a zero-island static script.
  // Mirrors @khazana/worker's client contract: POST `${endpoint}/event` with
  // { itemId, type, at, deviceId, dwellMs? }. Fully no-ops when endpoint is "".
  (function () {
    if (!endpoint) return; // graceful no-op without PUBLIC_WORKER_URL

    function deviceId() {
      try {
        var k = "khazana:deviceId";
        var v = localStorage.getItem(k);
        if (!v) {
          v = crypto.randomUUID();
          localStorage.setItem(k, v);
        }
        return v;
      } catch (_e) {
        return "anon";
      }
    }

    function emit(body) {
      try {
        var full = Object.assign({ at: new Date().toISOString(), deviceId: deviceId() }, body);
        var url = endpoint + "/event";
        var json = JSON.stringify(full);
        if (navigator.sendBeacon) navigator.sendBeacon(url, json);
        else fetch(url, { method: "POST", keepalive: true, headers: { "Content-Type": "application/json" }, body: json });
      } catch (_e) {
        /* never throw, never block navigation */
      }
    }

    // `open` on outbound feed/workshop clicks tagged data-track="open".
    document.addEventListener(
      "click",
      function (ev) {
        var el = ev.target && ev.target.closest ? ev.target.closest('[data-track="open"]') : null;
        if (!el) return;
        var id = el.getAttribute("data-item-id");
        if (id) emit({ itemId: id, type: "open" });
      },
      { capture: true },
    );

    // On a Read detail page: `open` immediately, `dwell` on unload.
    if (isRead) {
      var slug = location.pathname.replace(/\/$/, "").split("/").pop() || location.pathname;
      var start = Date.now();
      emit({ itemId: slug, type: "open" });
      window.addEventListener("pagehide", function () {
        emit({ itemId: slug, type: "dwell", dwellMs: Date.now() - start });
      });
    }
  })();
</script>
```

Both no-op identically without an endpoint.
